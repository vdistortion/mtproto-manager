## MTProto Proxy Manager 🚀

Удобный bash-скрипт для быстрого развертывания и управления MTProto-прокси серверами через Docker. Идеально подходит для создания индивидуальных прокси с поддержкой FakeTLS.

## Особенности ✨

- **Изоляция:** Каждый пользователь работает в отдельном Docker-контейнере.
- **FakeTLS:** Автоматическая генерация ee-секретов с маскировкой под популярные домены.
- **Умный выбор доменов:** Встроенные списки доменов для серверов в РФ и за рубежом.
- **Поддержка личных доменов:** Скрипт может генерировать ссылки как по IP-адресу, так и по вашему личному домену (домен хранится локально и не попадает в код).
- **Авто-MSS:** Скрипт сам настраивает правила iptables (TCPMSS) для стабильной работы.
- **Совместимость:** Обходит порты, зарезервированные панелью Marzban.
- **Бэкапы:** Удобный экспорт и импорт всех пользователей одной командой.

## Установка и запуск 🛠

Скрипт требует прав root (sudo). Поддерживаются Ubuntu, Debian и другие системы с apt.

### Шаг 1. Скачивание скрипта

Загрузите скрипт на свой сервер и выдайте ему права на выполнение:

```shell
wget https://raw.githubusercontent.com/vdistortion/mtproto-manager/main/mtproto.sh
chmod +x mtproto.sh
```

### Шаг 2. Первичная настройка

Выполните базовую настройку сервера (установит Docker, настроит ядро и сеть):

```shell
sudo ./mtproto.sh setup
```

(Во время настройки скрипт предложит ввести ваш личный домен для генерации красивых ссылок. Это необязательно — можно просто нажать Enter).

## Использование 📖

### Создание прокси

```shell
sudo ./mtproto.sh create <имя_пользователя> [домен] [порт]
```

Если не указать домен, скрипт выберет случайный из встроенного списка. Если не указать порт, скрипт автоматически найдет свободный, начиная с 8443.

Примеры:

```shell
sudo ./mtproto.sh create user1
sudo ./mtproto.sh create user2 google.com
sudo ./mtproto.sh create user3 ya.ru 9000
```

### Управление

- **Список всех прокси:** sudo ./mtproto.sh list (ссылки генерируются на лету, не сохраняясь в файлах)
- **Проверка статуса:** sudo ./mtproto.sh check (показывает, запущен ли Docker-контейнер для каждого пользователя)
- **Синхронизация и запуск:** sudo ./mtproto.sh sync (автоматически запускает или пересоздает контейнеры для всех пользователей из конфигов, если они были остановлены)
- **Восстановление списка:** sudo ./mtproto.sh rebuild (пересоздает индексный файл пользователей, полезно при сбоях)
- **Удаление прокси:** sudo ./mtproto.sh delete <имя_пользователя>

### Резервное копирование

- **Экспорт:** sudo ./mtproto.sh export (создаст .tar.gz архив в текущей папке)
- **Импорт:** sudo ./mtproto.sh import <имя_файла.tar.gz>

## Настройка региона 🌍

Откройте скрипт в любом редакторе (например, nano mtproto.sh) и измените переменную REGION в самом начале файла:

- REGION="GLOBAL" — для серверов в Европе/США (использует домены google.com, apple.com и т.д.)
- REGION="RU" — для серверов в России (использует ya.ru, vk.com и т.д.)

## Изменение личного домена 🌐

Если вы пропустили ввод домена при установке или хотите его изменить, просто запишите его в локальный файл:

```shell
echo "ваш-домен.com" > /root/.mtproto/domain.txt
```

Чтобы удалить домен и оставить только IP-ссылки:

```shell
rm /root/.mtproto/domain.txt
```
