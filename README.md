# 🚀 MTProto Proxy Manager (Docker + Auto-MSS)

Удобный bash-скрипт для развертывания и управления MTProto прокси-серверами в Docker на Ubuntu/Debian.

Скрипт автоматически настраивает ядро (BBR, MTU) и **динамически управляет правилами iptables (TCP MSS clamping)** "на лету". Вам больше не нужно вручную перезапускать настройку сети при добавлении или удалении пользователей — скрипт всё делает сам.

## ✨ Особенности
- 🐳 Работает на базе официального Docker-образа `telegrammessenger/proxy`.
- ⚡ Автоматически включает BBR и настраивает MTU для обхода блокировок и ускорения работы.
- 🛡️ Умное управление фаерволом: правила `iptables` добавляются при создании прокси и бесследно удаляются при его удалении.
- 📦 Встроенная система бэкапов: легкий экспорт и импорт всех пользователей при переезде на новый VPS.
- 🔗 Автоматическая генерация ссылок для подключения (`tg://` и `https://t.me/`).

## 🛠️ Установка и первый запуск

Скрипт необходимо выполнять от имени **root** (или через `sudo`).

**1. Создайте файл скрипта на сервере:**
```bash
nano mtproto.sh
```
*(Вставьте код скрипта, сохраните `Ctrl+O`, нажмите `Enter` и закройте `Ctrl+X`)*

**2. Сделайте скрипт исполняемым:**
```bash
chmod +x mtproto.sh
```

**3. Запустите первичную настройку сервера:**
*(Выполняется **один раз** на чистом сервере)*
```bash
sudo ./mtproto.sh setup
```

## 📖 Использование (Команды)

### ➕ Создание прокси
Создает новый контейнер, генерирует секрет, открывает порт и применяет правила фаервола.
```bash
sudo ./mtproto.sh create <имя_пользователя> [домен] [порт]
```
**Примеры:**
- `sudo ./mtproto.sh create ivan` *(создаст прокси для ivan на порту 443 с доменом ya.ru)*
- `sudo ./mtproto.sh create maria google.com 8443` *(создаст прокси на порту 8443 с доменом google.com)*

### 📋 Список пользователей
Выводит список всех активных прокси, их порты, домены и готовые ссылки для подключения.
```bash
sudo ./mtproto.sh list
```

### 🗑️ Удаление прокси
Останавливает контейнер, удаляет конфиг и **автоматически очищает правила iptables** для этого порта.
```bash
sudo ./mtproto.sh delete <имя_пользователя>
```
**Пример:**
- `sudo ./mtproto.sh delete ivan`

## 📦 Переезд на новый сервер (Бэкап и Восстановление)

Скрипт позволяет перенести всех пользователей, их порты и секреты на новый VPS в пару кликов. Ссылки у пользователей **не изменятся** (кроме IP-адреса, если вы не используете привязку к домену).

### Шаг 1: Экспорт (на старом сервере)
```bash
sudo ./mtproto.sh export
```
В текущей папке появится архив вида `mtproto_backup_20231025_120000.tar.gz`. Скачайте его на свой компьютер или сразу передайте на новый сервер.

### Шаг 2: Подготовка нового сервера
Загрузите на новый сервер скрипт `mtproto.sh` и архив с бэкапом.
```bash
chmod +x mtproto.sh
sudo ./mtproto.sh setup
```

### Шаг 3: Импорт (на новом сервере)
```bash
sudo ./mtproto.sh import mtproto_backup_20231025_120000.tar.gz
```
**Что произойдет:** Скрипт распакует конфиги, заново создаст Docker-контейнеры для каждого пользователя и **автоматически восстановит все правила iptables** для их портов. Прокси сразу начнут работать!

## 📁 Структура файлов
Все данные скрипта хранятся в директории пользователя, от имени которого он запускается (обычно `/root/.mtproto/`):
- `users.conf` — общий список активных пользователей.
- `<username>.conf` — индивидуальный файл конфигурации пользователя (содержит порт, секрет, ссылки).
